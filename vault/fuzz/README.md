# vault/fuzz — Fuzzing the v4 Vault Decode Path

This directory contains the [`cargo-fuzz`](https://rust-fuzz.github.io/book/cargo-fuzz.html)
setup for `botwork-vault`.  It exercises three code paths that parse
untrusted input:

| Exercise | Code path |
|----------|-----------|
| Full file decode | `Vault::unlock` → header checks → HKDF → ChaCha20-Poly1305 AEAD → `postcard::from_bytes::<VaultContents>` |
| Direct postcard decode | `postcard::from_bytes::<VaultContents>` without a key |
| Per-entry envelope decode | `postcard::from_bytes::<EntryEnvelope>` → `open_entry` → `unwrap_dek_under_master` → per-entry AEAD |

**Invariant:** no input may cause a panic or abort.  Every malformed
input must map to a structured `VaultError` variant.

---

## Prerequisites

```sh
# Install cargo-fuzz (requires nightly Rust)
cargo install cargo-fuzz

# Make sure the nightly toolchain is available
rustup toolchain install nightly
```

---

## Running the fuzz target

All commands must be run from the **vault crate root**
(`botwork-extra/vault/`), _not_ from `vault/fuzz/` itself.

### Quick smoke-run (a few seconds)

```sh
# From vault/
cargo +nightly fuzz run fuzz_open_contents -- -max_total_time=30
```

### Continuous run (stops after finding a crash or until you kill it)

```sh
cargo +nightly fuzz run fuzz_open_contents
```

### Running with the seed corpus

The `corpus/fuzz_open_contents/` directory ships three seed inputs:

| File | Description |
|------|-------------|
| `valid_v4_vault.bin` | A fully-valid v4 vault with one entry (270 bytes) |
| `valid_v4_vault_empty.bin` | A valid v4 vault with no entries (62 bytes) |
| `valid_contents_postcard.bin` | A valid postcard-serialised `VaultContents` (12 bytes) |

`cargo fuzz run` picks up the corpus automatically from the conventional
location.  To point it explicitly:

```sh
cargo +nightly fuzz run fuzz_open_contents fuzz/corpus/fuzz_open_contents
```

### Reproduce a specific input

```sh
cargo +nightly fuzz run fuzz_open_contents -- path/to/crash/or/artifact
```

### Coverage report

```sh
cargo +nightly fuzz coverage fuzz_open_contents
```

## Regenerating the seed corpus

The seed files were generated by the following Rust snippet using the
vault's public API.  Re-run it any time the v4 format changes:

```rust
use std::fs;
use botwork_vault::{SecretEntry, SecretKey, SecretKind, Vault, VaultContents};

const EXPORT_KEY: &[u8; 64] =
    b"deterministic-export-key-bytes-for-fuzz-target-AAAAAAAAAAAAAAAAA";

let dir = tempfile::tempdir().unwrap();
let root = dir.path().join("vault");
let now = chrono::Utc::now().timestamp();

// valid_v4_vault.bin — a vault with one entry
let mut vault = Vault::create(&root, EXPORT_KEY, 1).unwrap();
vault.put_secret(
    SecretKey { service: "github".to_string(), name: "pat".to_string() },
    SecretEntry {
        kind: SecretKind::ApiKey,
        value: b"ghp_corpus_seed_value".to_vec(),
        created_at: now, updated_at: now,
        last_used_at: None, tags: vec![], allowed_consumers: vec![],
    },
).unwrap();
vault.lock();
fs::write("corpus/fuzz_open_contents/valid_v4_vault.bin",
          fs::read(root.join("vault.botwork")).unwrap()).unwrap();

// valid_v4_vault_empty.bin — empty vault
let dir2 = tempfile::tempdir().unwrap();
let root2 = dir2.path().join("vault");
Vault::create(&root2, EXPORT_KEY, 1).unwrap();
fs::write("corpus/fuzz_open_contents/valid_v4_vault_empty.bin",
          fs::read(root2.join("vault.botwork")).unwrap()).unwrap();

// valid_contents_postcard.bin — raw postcard VaultContents bytes
let contents = VaultContents::new(now);
fs::write("corpus/fuzz_open_contents/valid_contents_postcard.bin",
          postcard::to_allocvec(&contents).unwrap()).unwrap();
```

## Companion: property/corruption tests

`vault/tests/corruption.rs` contains a `proptest`-based corruption suite
that runs under normal `cargo test` (no nightly required):

```sh
cargo test -p botwork-vault --test corruption
```

These tests generate a valid vault, apply random single-byte flips,
region overwrites, truncations, and arbitrary-byte inputs, and assert
the open path returns only structured `VaultError` variants.
